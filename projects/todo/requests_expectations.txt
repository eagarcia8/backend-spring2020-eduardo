jQuery Post Request Syntax

$.post(addressInString, objectToSend, callbackToRunWhenWeGetAResponse(dataWeGotFromBackend) {});


The /newNote Route allows the front end to submit a note to save. In return the server will tell us if the note was saved or not.

Object to send from front-end for /newNote
{
	note: String (The note text to save)
	author: Null or String (The user or owner of the note)
}

Object to send to front-end for /newNote
{
	saveStatus: Number (0 for saved, 1 for error)
}

The /readNotes Route allows the front end to get all notes saved on the back-end. We just need to send an empty object, and expect an object with all the notes in it.

Object to recieve from front-end for /readNotes
{
}

Object to send to front-end for /readNotes
{
	notes: Array
}


The /deleteNote Route allows the front end to request a note to be deleted by sending the identifying information of that note, it should get back an object that says that the note was successfully deleted or not.


Object to recieve from front-end for /deleteNote
{
	create_date: Number,
	author: String
}

Object to send to front-end for /deleteNote
{
	deleteStatus: Number (0 for successfully deleted, 1 for not found, 2 for found but not deleted)
}

The /updateNote Route allows the front-end to send information to the back-end to change the note string of a specific note.

Object to send to the back-end for /updateNote
{
	updated_note: String
	create_date: Number,
	author: String
}

Object to send to the front-end for /updateNote
{
	updatedStatus: Number (0 if found and updated, 1 if not found, 2 if found not updated)
}


The /markComplete route sets a specific note to completed status, it should return confirmation of the request.

Object to send to back-end for /markComplete
{
	create_date: Number,
	author: String
}

Object to send from back-end for /markComplete
{
	markedComplete: Number (0 if successfully marked, 1 if not found, and 2 if found but not marked)
}
